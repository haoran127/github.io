评估后Broadcom世界中的VMware替代方案

发表于 2024-05-14 更新于 2024-09-18 阅读次数：

随着 Broadcom 收购 VMware，许多人都想知道 VMware 服务的下一步是什么，包括支持和更新。这使得研究其他虚拟化选项成为任何依赖 VMware 技术进行设置的人必须做的事情。

我们将在本文中深入探讨 VMware 替代方案，旨在为您或您的团队提供所需的知识，以便做出适合您独特需求的明智选择，从权衡成本和检查支持级别到弄清楚这些系统的复杂程度。我们将研究硬件兼容性，重点关注进行切换的容易程度。我们将其分为三个部分：迁移考虑因素、VMware 替代方案和 VMware 迁移工具

从 VMware 迁移时要考虑的因素

成本可能是组织必须考虑的更重要的因素之一，因为与之相关的是复杂的许可和高昂的使用费。据 Gartner 称，近 50% 的 IT 专业人员将成本视为寻找 VMware 替代方案的主要因素。还有很多人担心持续的产品支持，特别是对于较小的帐户。

找到更易于使用的定价模型来简化成本计算并降低总体费用至关重要。这可能涉及寻找基于节点定价的解决方案，从而无需计算内核、RAM 和容量。

支持是另一个关键方面。鉴于基础设施软件在数据中心中的重要作用，仅仅拥有基于社区/表单的支持或仅电子邮件支持有时是不够的，具体取决于组织的规模或用例。许多组织应该选择能够提供强大、以人为本的支持和快速响应时间的解决方案，以确保平稳和持续的运营。例如，勒索软件和暴力攻击等紧急情况需要有效且立即的解决方案，而纯在线支持系统可能无法提供该解决方案。

软件的复杂性也是一个问题。虽然 VMware 可能难以管理，但许多开源替代方案甚至更加复杂，需要专家来启动和运行一切。简化运营、提高数据效率和弹性以及最大限度缩短管理时间的解决方案对于许多组织来说至关重要。例如，采用“超融合”方法的替代方案可能是简化运营并减轻 IT 人员负担的一种方法；计算、网络和存储功能不仅融合到单一解决方案中，而且还经过高度优化和简化。

硬件选择和迁移的便捷性虽然对于成功过渡是必要的，但却是您应该考虑的最后一个重要事项。组织经常面临供应商特定的交钥匙捆绑包（仅满足特定需求的捆绑包）或硬件兼容性列表的限制。硬件要求最低的灵活解决方案可提供显着的成本、性能和可扩展性优势。顺利的迁移过程也至关重要，特别是对于对复杂的数据中心更新持谨慎态度的公司。能够将虚拟机无缝复制到新目标直至完全集成到生产中的解决方案是理想的解决方案，因为它们可以最大限度地减少中断并促进成功迁移。

最终，在 VMware 收购之后找到合适的虚拟化解决方案需要仔细考虑所有这些因素，因为这对于确定符合组织特定需求和限制的虚拟化平台至关重要。

根据我们的反馈和评论 不和谐社区，我们编制了一份 VMware 替代方案列表，包括优点和缺点。继续阅读了解更多详细信息。

VMware 替代方案

本节介绍前五名 VMware 替代方案。我们尝试在适当的情况下添加优点和缺点，但最终的决定应该基于组织对成本、支持、实施、易用性和管理的要求。

Microsoft Hyper-V：具有强大支持的集成解决方案

Microsoft Hyper-V 代表了一个由可靠支持补充的集成解决方案，使其成为那些已经在 Microsoft 生态系统中运行的用户的可行选择。 Hyper-V 包含在 Windows Server 中，可以潜在地降低已投资于 Microsoft 基础设施的组织的成本。由于它是一家大公司的一部分，它还受益于全面的支持（例如直接的、一对一的协助），这对于解决关键问题极其重要。

此外，Hyper-V 的设计对于习惯 Windows 环境的个人来说是用户友好的，尽管它可能会给那些不太熟悉操作系统的人带来学习曲线。尽管它主要适用于以 Windows 为中心的设置，但它具有相当大的硬件兼容性。 Hyper-V 还通过专用工具促进 VMware 的更顺畅的迁移过程，使其成为寻求平稳过渡的组织的一个令人信服的选择。

还值得注意的是，微软还为那些想要融合产品的人提供 Azure Stack HCI。我们在这个平台上做了很多工作， 最近通过 DataOn。还有一些简洁的产品，例如 托管SQL 如果您正在考虑使用 Microsoft/Azure 生态系统，那么这些内容值得一看。

Microsoft Hyper-V 功能包括：
Microsoft Hyper-V 实时迁移 能够在 Hyper-V 主机之间移动正在运行的虚拟机，而不会导致停机。实时迁移的主要好处是灵活性；运行的虚拟机不依赖于单个主机。用户可以在退役或升级之前耗尽特定主机的虚拟机。与 Windows 故障转移群集配合使用时，实时迁移允许创建高可用性和容错系统。

Hyper-V 提供了一系列 Linux 发行版支持 用于虚拟化 Linux 工作负载的更通用的平台。 Hyper-V 支持 Linux 和 FreeBSD 虚拟机的模拟设备和 Hyper-V 特定设备。使用模拟设备运行时，无需安装其他软件。

Hyper-V 支持嵌套虚拟化，允许用户在 Hyper-V 虚拟机 (VM) 内运行 Hyper-V。随着硬件的进步，嵌套虚拟化的用例已经扩大。

当然，与许多替代解决方案一样，也存在一些弱点。最大的缺点可能是 Hyper-V 仅与 Windows Server 兼容。运行其他操作系统的用户还需要迁移到 Windows 平台。 VMware 包含比 Hyper-V 更多的第三方工具，这增加了设置和管理的复杂性。

Citrix Hypervisor：专注于可扩展性和性能

思杰Hypervisor 以注重可扩展性和性能而闻名，使其成为 VDI（虚拟桌面基础设施）部署的不错选择。对于许多用例来说，它是一种经济高效的解决方案，但可能会产生额外的许可成本，具体取决于部署的规模和复杂性。此外，Citrix 还提供全面的支持，这对于管理大规模、性能关键的部署至关重要。虽然该平台在可扩展性方面表现出色，但它引入了一定程度的管理复杂性，这意味着您可能需要现有的专业知识才能有效运营。

在硬件兼容性方面，Citrix Hypervisor 支持一系列配置。这使得它适合不同的 IT 环境，例如具有混合硬件设置、希望整合虚拟化平台的组织。当涉及从其他解决方案迁移时，Citrix 提供了促进该过程所需的工具和支持。但是，潜在用户应该知道，过渡到 Citrix Hypervisor 可能比简化的虚拟化解决方案更加复杂。因此，这可能需要仔细规划和执行才能成功迁移。

Citrix Hypervisor 加载的功能包括：
与 Hyper-V 类似，Citrix Hypervisor 支持服务器之间的实时 VM 迁移，以确保 VM 在发生硬件故障或其他异常情况时可用。

Citrix Hypervisor 专为实现高性能和可扩展性而设计，每台主机支持多达 288 个内核和 12TB RAM。虚拟机管理程序支持包括安全启动、安全 VM 迁移以及与 Active Directory 的集成。 Citrix Hypervisor 被认为是 VMware 的经济高效替代方案，提供不需要许可证费用的功能。

尽管 Citrix Hypervisor 是 VMware 替代方案的首选，但在开始使用之前还需要考虑一些方面。与 Hyper-V 一样，与 VMware 相比，Citrix Hypervisor 的第三方支持有限。这意味着集成和工具可用性可能会减少。 Hypervisor 也比 VMware 更难设置和管理，并且缺少 VMware 用户喜欢的一些高级功能。

红帽虚拟化：开源灵活性和企业级支持
VMware 的另一个值得注意的替代方案是 红帽虚拟化（RHV），一个基于 KVM 管理程序的开源虚拟化平台，专为需要强大的虚拟化解决方案而不受专有软件固有限制的企业而设计。这对于旨在绕过这些专有选项典型的高额许可费用的企业特别有利，因为其开源框架可确保更可预测的定价并可能降低总拥有成本。

RHV 还提供红帽提供的优质企业级支持。对于关键系统依赖可靠援助并需要迅速解决复杂问题的组织来说，这一点至关重要。此外，尽管 RHV 具有全面的功能集，但它的设计易于使用，具有基于 Web 的管理界面，旨在简化虚拟环境的管理。也就是说，对红帽生态系统的一些先验知识有利于提高整体可用性。

红帽虚拟化还具有广泛的硬件兼容性。这意味着它是一种灵活的解决方案，可以适应各种 IT 环境，而无需依赖于特定的硬件供应商。此外，由于 RHV 专门为此目的而设计了工具和服务，因此它支持从 VMware 等平台进行更顺畅的迁移。

尽管 RHV 提供可扩展性、支持数千个虚拟机、支持实时迁移、高度安全并且与红帽企业 Linux 集成，但仍有一些领域可能会引起一些担忧。最令人担忧的可能是红帽虚拟化于 2022 年 2024 月进入维护支持，并于 31 年 2026 月结束，并于 XNUMX 年 XNUMX 月 XNUMX 日进入延长生命周期阶段。

其他重要的考虑因素包括陡峭的学习曲线和有限的第三方支持。很容易在这些替代方案中看到一个共同的主题。

Docker 和 Kubernetes
尽管有些人可能不同意，但 Docker 和 Kubernetes 已经变得足够强大，可以考虑在各种规模的企业中使用。 VMware 长期以来一直是企业虚拟化的标准，而颠覆者 Docker 和 Kubernetes 都有自己独特的优势，使其适合各种工作负载和用例。

码头工人 容器以其速度和效率而闻名，因为它们可以在几秒钟内启动，并且与需要加载整个操作系统的虚拟机 (VM) 相比，它们占用的资源较少。这使得扩展和复制容器比虚拟机更容易、更快捷。 Docker 的轻量级特性允许快速部署和管理应用程序，使其特别适合服务频繁更新和扩展的微服务架构。

Docker 允许组织绕开与传统虚拟化解决方案相关的高昂许可费用，根据容器数量而不是核心或 RAM 等硬件规格进行收费。

Kubernetes另一方面，擅长跨分布式环境管理容器化应用程序。它是一个编排平台，可自动执行应用程序容器的部署、扩展和操作。其跨不同环境（例如公共云和本地）大规模管理容器的能力是其广泛采用的关键原因之一。 Kubernetes 可以提高应用程序管理的生产力，确保应用程序的可靠性，而不会在更新期间停机，并扩展基础设施，而不会产生不必要的成本。它还因其灵活性、兼容性和强大的社区支持而受到称赞。

尽管 Kubernetes 是免费的，但用户应该为云平台上的托管服务计划额外费用。

当将 Docker 和 Kubernetes 与 VMware 的稳健性进行比较时，这不仅与企业规模有关，还与 IT 环境的架构和特定需求有关。 VMware 通过 Tanzu 等产品将 Kubernetes 集成到其产品组合中，这表明了 VMware 对容器化以及传统虚拟化的承诺。这种集成为管理虚拟机和容器提供了简化的体验，并提供了操作的一致性和简单性。

从本质上讲，Docker 和 Kubernetes 对于希望在其应用程序中利用容器的企业来说是强大的选择。它们不仅适用于中小型企业；它们还可以支持大规模、复杂的应用程序和工作负载。 Docker、Kubernetes 和其他替代方案之间的选择很大程度上取决于组织的具体要求、现有基础设施和战略目标。每种技术在现代 IT 环境中都有自己的位置，许多企业在利用容器化和传统虚拟化的混合方法中发现了价值。

这些基于容器的解决方案的迁移过程各不相同。虽然云原生应用程序可以平滑过渡，但传统应用程序可能需要大量重构。

提名
普罗克斯莫克斯: proxmox ve （Virtual Environment）是一个开源的服务器虚拟化管理平台。它将 KVM 管理程序和 LXC、软件定义存储和网络功能集成在单一平台上。企业支持是 Proxmox 最大的问题，尽管我们听说一些集成商正在研究减轻这种问题的方法。

Oracle VirtualBox：VirtualBox 是一个免费的开源托管虚拟机管理程序，用于 x86 虚拟化。它专为服务器、桌面和嵌入式使用而设计。

努坦尼克斯 AHV：Nutanix AHV 是 VMware vSphere 的替代方案。 AHV（Acropolis Hypervisor）和 Nutanix 企业级虚拟机管理程序包含在企业云操作系统中。 Nutanix 历来被认为是昂贵的，现在 AHV 被视为更便宜的 VMware 替代品，这有点幽默。

AWS：Amazon Web Services 提供一套云计算服务，被认为是 VMware 的替代品，特别是其 EC2 实例可在 AWS 云中提供可扩展的计算能力。

谷歌云: 与AWS类似，Google Cloud是Google提供的一套云服务，是VMware的替代品。它提供与 VMware 相同类别的所有服务，例如计算、存储和网络。

开栈：Openstack 是一个开源云计算平台，通常部署为 IaaS（基础设施即服务），使其成为 VMware 的替代品。它通常用于在云计算环境中设置和管理大量虚拟专用服务器。

星风：StarWind 提供用于 VMware 替代品和 VMware 迁移的产品。他们有自己的超融合基础设施平台，可以被视为VMware的替代品，他们还提供StarWind V2V Converter，这是一个免费工具，用于将虚拟机从一种格式转换为另一种格式，可用于VMware迁移。

XCP-ng：XCP-ng（Xen Cloud Platform – New Generation）是一个基于XenSource XenServer的开源虚拟化平台。它使用户能够像 VMware 一样托管、管理和部署虚拟机。

边缘网: 我们拍了一张 深入了解 Verge.io 2022 年，我们对该产品印象深刻。我们多次鼓励他们提供社区版本，但他们尚未接受，但对于规模较大的组织来说，这个版本值得一试。

规模计算: Scale was another candidate for a 产品深度挖掘。规模计算取代了现有的基础设施，使企业能够在最接近数据创建和使用位置的集中式数据中心之外运行应用程序和处理数据。规模也启动了战略 VMware 淘汰和替换 促销活动旨在支持合作伙伴和客户从 VMware 过渡到 Scale Compute 平台。

KubeVirt: KubeVirt is an open-source project that extends Kubernetes to manage VMs alongside containers, offering a unified platform for both cloud-native and traditional applications. It is designed to streamline the operational management of diverse workloads, enhance flexibility and portability across different environments, and benefit from robust community support. Despite its benefits, KubeVirt introduces a learning curve for Kubernetes newcomers, potential performance overheads, and limited integrations. Explore the KubeVirt User Guide 了解更多信息。

这些平台用于创建和管理虚拟机和容器，提供与 VMware 类似的功能，但具有不同的功能、定价和开源可用性。它们之间的选择通常取决于特定的业务需求、技术要求和预算考虑。

当放弃 VMware 时，根据已确定的关键因素权衡这些替代方案至关重要。 Docker 和 Kubernetes 非常适合构建经济高效、可扩展的解决方案，Hyper-V 适合在 Windows 环境中无缝集成，而 Citrix Hypervisor 则适合稳健的大规模部署。选择应符合您组织的预算、所需的支持级别、技术复杂性、硬件偏好以及高效迁移过程的需求。

VMware 迁移工具
选择合适的平台从 VMware 进行迁移后，下一步就是选择最适合您的数据的迁移工具。有多种工具可供使用，每种工具都具有独特的优势，旨在满足不同的需求。选择能够与您选择的平台顺利集成并精心满足您的特定迁移要求的工具至关重要。

我们探索了 Microsoft Hyper-V、Red Hat Virtualization (RHV) 和 Citrix Hypervisor 作为 VMware 的可行替代方案，并指出它们不仅具有作为虚拟化平台的功能，而且还可以作为实用的迁移工具。让我们扩大讨论范围，研究最适合特定场景和组织要求的其他迁移工具。

从知名度和功能来看，排名前两位的 VMware 迁移工具实际上都来自 VMware。

VMware vSphere Replication：VMware vCenter 的扩展，提供基于管理程序的虚拟机复制和灾难恢复。
VMware HCX：一个应用程序移动平台，旨在简化跨数据中心和云的工作负载迁移、工作负载重新平衡和业务连续性。
Carbonite Migrate：允许您以最小的风险和接近零的停机时间在任何距离内迁移物理、虚拟和云工作负载。它提供自动迁移，不断复制字节级别的更改，以保持较低的传输时间。
CloudEndure Migration：该工具被 AWS 收购，可简化、加速和自动化大规模迁移到 AWS。 CloudEndure Migration 不断将源计算机复制到您的 AWS 账户中的暂存区域，而不会导致停机。
Zerto：提供 IT 弹性平台，该平台结合了灾难恢复、备份和工作负载移动性，有助于跨不同环境迁移和保护 VMware 工作负载。
Turbonomic：现已成为 IBM 的一部分，提供迁移工具，帮助自动化和优化迁移规划，使应用程序能够跨任何云或数据中心自我管理资源。
Microsoft Azure 站点恢复：虽然通常用于灾难恢复，但它还支持迁移，并可以自动将虚拟机复制到 Azure，从而简化迁移过程。
vCenter Converter：一种 VMware 解决方案，可将本地和远程物理机转换为虚拟机，无需任何停机。它支持多种源物理机，包括Windows和Linux桌面和服务器版本。
这些工具中的每一个都具有不同的特性和功能，可能使它们更适合特定的迁移场景，因此必须根据组织的需求对其进行评估。始终确保兼容性，并确保该工具满足您环境的安全性和合规性要求。有关最新的特性和功能，以及有关哪种工具最适合您的需求的指导，请访问供应商的官方网站或联系他们的销售团队。

结论
这里讨论的每个 VMware 替代方案都有自己的一组功能、优点和注意事项。然而，在决定虚拟化解决方案之前，组织必须评估其特定需求，例如运营规模、预算限制和基础设施偏好。尽管如此，我们强烈建议利用试用期并对最感兴趣的选项进行详细评估。这样做将极大地帮助您做出符合您需求的明智决定。
